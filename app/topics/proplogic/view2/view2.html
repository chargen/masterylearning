Tree-based formula evaluation

Given the following variable assignments:

  • `a rarr 0`
  • `b rarr 1`
  • `c rarr 1`
  • `d rarr 1`

The following tree represents the boolean formula `(a ^^ b) vv (c o+ D)`. Please assign the values of the atoms and operators to the respective boxes.

`vv`
`^^`
`o+`
`a`
`b`
`c`
`d`

Answer is correct.

Answer is wrong. Please take a look at the following truth table for the `vv` operator:

`vv` 0 1
0 0 1
1 1 1


Storyboard

Goal: This exercise demonstrates the evaluation of propositional logic in tree form. The advantage over the exercise type 1 is, that the operator precedence and parentheses are already clear through the tree form, so we can check directly if the student is able to evaluate the logical operators.
The disadvantage is, that we have already evaluated parentheses and operator precedences and thus this exercise type is somewhat easier then the other ones.

Exercise Generation and Variations: When generating this exercise, we can vary the following elements:

  • Number and letters of variables
  • Set of operators (e.g. `vv, ^^, o+, ...`)
  • Nesting depth, number of terms
  • We could also choose to generate only certain forms of formulas such as DNF, CNF, Horn clauses, etc.

Usage: Please type `1` into the box of the `vv` operator and press check answer. You will the that the truth table for the `vv`-operator is shown since the expected answer is 0.

Data collection: For each operator that was evaluated incorrectly we keep a record of how many times the student made a mistake with that operator. Then we can adjust further questions to explicitly cover this operator.

Improvements: In the final version we plan to draw the tree on the left in a more graph-like fashion by using an appropriate drawing library.
